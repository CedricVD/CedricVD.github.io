Project Mastermind

Ons eerste project had als doel het spel MasterMind na te maken.
Samen met 3 andere studenten, hadden we enkele maanden tijd om een werkende versie van het populaire spel MasterMind te maken.

Om dit project tot een goed einde te brengen, hebben we het opgedeeld in 7 afzonderlijke functionaliteiten. Deze functionaliteiten hebben we stap na stap uitgewerkt door eerst een ruwe analyse te maken van de vereisten, om deze nadien verder te definiëren.

Voor de implementatie maakten we gebruik van Java. Dit was onze eerste programmeertaal die we geleerd hadden, en dit dus ook de enige optie. Het voordeel van Java was dat deze taal zeer gemakkelijk te hanteren is. Naar het einde toe ondervonden we wel wat problemen bij het gebruik van Java, namelijk de moeilijkheid om alles gestructureerd te houden.

De eerste functionaliteiten gingen zonder problemen, maar na verloop van tijd merkten we dat onze oorspronkelijke implementatie niet robuust genoeg was en dus voor een deel herwerkt moest worden. Hierdoor ontstonden er redelijk wat problemen die moesten opgelost worden. Het probleem dat we hierbij hadden, was dat we de deadline aan het naderen waren. We hebben toen besloten dat we het team in 2 zouden splitsen. Het ene team zou zich focussen op verdere implementatie, terwijl het andere team de ontstane bugs zou oplossen. Hierdoor hadden we regelmatig last van miscommunicatie maar in het einde is het ons toch gelukt om een werkende applicatie af te leveren.


Taijitan

Het tweede project was in opdracht van Taijitan Yoshin Ryu, een Gentse Jiu Jitsu vereniging.
De opdracht van deze vereniging was het maken van een webtool die beheerd kan worden via een aparte administratortool.
Hiermee zouden leden dan lessen kunnen raadplegen volgens hun graad en hun aanwezigheden in de les registreren op een efficiëntere manier.

Dit was ons eerste project voor een echte opdrachtgever. Het was tevens ook ons eerste project dat bestond uit 2 afzonderlijke projecten. Langs de ene kant was er de webtool, maar langs de andere kant was er ook de administratortool. Voor dit project werd er gewerkt in iteraties. Elke twee weken was er een contactmoment waarop we vragen konden stellen in verband met bepaalde vereisten. Dit was zeer handig aangezien er doorheen het project regelmatig nieuwe eisen bij kwamen en bepaalde eisen werden vervangen. De webtool werd gemaakt in ASP.NET, de administratortool werd gemaakt in Java. ASP.NET was iets dat we nog maar onlangs geleerd hadden, en dit was dus nog wat onwennig. Na de eerste weken ging het steeds beter en hebben we veel bijgeleerd.

De administratortool was ook iets nieuw voor ons. We waren het ondertussen al gewoon om in Java te werken, maar aangezien er al data in onze databank zat door het maken van de webtool, moesten we deze data er ook weer uit halen. In de lessen hadden we nog nooit met een SQL Server databank gewerkt, dus om dit goed te laten verlopen hebben we zeer veel opzoekwerk moeten doen.

Onze applicatie was goed. Het was niet perfect, maar ook zeker geen teleurstelling. Onze grootste hindernis was het aanpassen van de databank. Doordat we allemaal een andere versie hadden, of zelfs andere software, hadden we ook verschillende instellingen. In de toekomst zal ik er zeker op letten dat we allemaal dezelfde setup gebruiken om zo deze 'domme' fouten te vermijden.


Mobile game: Horizon Cruiser

Horizon Cruiser is mijn persoonlijk project waar ik in de herfst van 2018 aan begonnen ben.
Het is een spel gemaakt in Unity, dat gebaseerd is op Temple Run en Subway Surfer. Het spel maakt gebruik van kleurrijke neon elementen in een 80's thema.

Ik ben met dit project begonnen uit interesse in Unity. Ik had hier nog niets van ervaring mee, maar het ontwikkelen van games was iets wat mij altijd al aansprak. Het thema van het spel is zeer toevallig ontstaan. Ik wou iets maken dat niet te moeilijk was aangezien ik nog geen ervaring had in Unity. Subway Surfer maakt gebruik van een zeer gemakkelijke input, dus ik wou mij hier op baseren. Na verloop van tijd zijn er meer en meer elementen aan het spel toegevoegd.

Het spel is momenteel nog in ontwikkeling. De persoon die zich bezig hield met de 3D sprites is momenteel op buitenlandse stage, waardoor het project uitgesteld is.


Klimaatmobiel

Het klimaatmobiel-project was het laatste groepsproject tijdens mijn opleiding. Het project werd opgesteld op aanvraag van het GO-onderwijs. De applicatie bestond uit twee afzonderlijke toepassingen. Er werd gebruik gemaakt van een Angular toepassing, een SQL Server database en een Android Studio project, gebruikmakende van Kotlin.

Dit project was het eerste 'large-scale' project waar ik ooit heb aan gewerkt. Aangezien niemand van de groep ervaring had met Android-development, had het project een aantal bugs die we dienden op te lossen. Door de omvang van het project, waren er ook meer teamleden bij betrokken. Het aantal teamleden zou er voor zorgen dat het werk sneller vooruit ging, maar dit bleek lastiger dan verwacht aangezien het niet evident was om alles te coördineren. Om het coördinatieprobleem om te lossen werd de groep in twee gesplitst. Terwijl de ene helft aan het Angular gedeelte werkte, werkte de andere helft aan het Android gedeelte. Er werd regelmatig gewisseld, zodat iedereen op de hoogte was van de wijzigingen in beide delen. Dit was ook noodzakelijk om iedereen up-to-date te houden op het vlak van de inhoud van de database.

Het resultaat van dit project was zeer positief. Op een aantal kleine bugs na, werkte alles zoals verwacht. De klant was zeer tevreden met het resultaat en de code was ook van zeer goede kwaliteit.
